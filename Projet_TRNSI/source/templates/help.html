Help

Cette page rassemble la documentation pratique du site Three.

Fonctionnement global

    Page Donnes

  • La page Donnees sert a creer les tables, les colonnes, les objets, les unites et les inversions.
  • ajout rapide : sert à ajouter des objets rapidements (sans blague!)
  • Table d'unités : Ce sont les unités pour la simulation, elles servent à faire les conversion
  • Table d'inversion : Table dans laquelle on stock les inversions. La simulation comprend que l'inverse de la valeur 0 est la valeur 1 et réciproquement.

    • Page Liaisons

  • La page Liaisons sert a relier les objets et a regler la conservation.
  • 2 types de liasons "=>" et "<=>" Ex: A=>B veut dire que A est lié à B mais pas l'inverse. Contrairement à A <=> voir synthaxe dans la bibliotheque Frank SQL
  • Elle affiche les liaisons sous forme de graphe.
  • Une liaison est toujours constitué d'une probabilite p comprise dans [0;1] de 1 par défaut
  • Elle peut aussi avoir une conservation c qui définit comment va être conserver la valeur transmise dans le réseau

    • Page Composants

  • La page Composants gere les objets composes et les fusions.
  • Quand un objet A devient composants d'un autre objet B, soit A(B) on obtient une liaison A => B
  • On représente l'objet et ses composants sous forme d'abre (on peut définir le nombre max de branche visible)
  • La fusion est une opération qui permet de faire de créer un nouvelle objet à partir d'autre objets qui deviennent ainsi ses composants, c'est comme une compo classique mais on crée l'objet. Pour la fusion, on peut quantifié sans mettre de ()

    • Page Simulation

  • La page Simulation applique les evenements et les paternes.
  • Un événement est de la forme Si ... Alors Faire ... avec une proba de ... à partir du tour ...
  • Un événement dépend d'une cause et d'une conséquence, la cause est construite sous forme de cible + observation et la conséquence sous la forme cible + action et ensuite après on ajoute des paramètre tel que la proba p de 1 par défaut, la propagation (en gros le nombre de sommets du graphe qui vont être impacté par l'évennement) écrire all pour tous les choisir, et l'arrivée dans la simulation (de 0 par défaut)
  • Les types d'Actions : Inversion (on inverse la valeur en utilisant la table d'inversion), Opération (opération classique style +8 -7 *0.58 /2), Changer : change la valeur cible par une autre
  • Les paternes sont des action qu'ont effectuent à une fréquences, si on précise pas l'unité cette fréquence est en tour
  • Quand on simule on choisit le nombre de tour que l'on souhaite observer et la valeur d'un tour, exemple 1tour = 6s et tu choisis 10 tours donc tu verra l'évolution de la simulation sur 1min, en 10 tours.
  • On choisit aussi un objet dans la liste des objets. Dans l'ajout rapide, vous pouvez mettre objet(colonne1 ; colonne2...); objet2(colone1) pour cibler les objets que vous souhaitez observer dans la simulation. Attention le graphique affiche en ordo. seulement la première colonne rentrée
  • On obtient donc un journal tours par tours avec le temps + un graphique

    • Page Code

  • La page Code execute du Frank SQL.
  • Voir la Bibliotheque Frank SQL

    • Page Map

  • La page Map enregistre les coordonnees geographiques d'un objet.
  • On a choisit la map du monde pour l'exemple, mais ce n'est pas obligatoire, en effet l'espace créer est virtuelle et sert surtout à rendre "reel" ou "existant" un objet, et à bien d'autre action/logique dans un espace
  • On peut visionner des chemins créer dans la page Code en Frank SQL

action :
op : operation sur une valeur numerique
change : remplace la donnee par une autre
inv : inversion de la donnee (table d'inversion)

variables :
p : probabilite comprise dans [0 ; 1]
c : conservation, n represente la valeur de l'operation effectuee sur l'objet precedent
vd : valeur par defaut, exemple : vd=0 signifie que la valeur par defaut de la colonne est 0
obj : objet
col : colonne
tab : table
f : frequence
x : position horizontale
y : position verticale

On separe les variables avec ","
On separe les elements avec ";"

types :
texte : texte
int : entier
float : reel

Synthax liaisons :
Elles se font dans le bloc obj{} ou dans l'ajout rapide de liaisons.

Soit les objets : A, B, C, D

A=>B signifie que si A est touche par un evenement ou un paterne, B peut l'etre aussi selon la propagation.
A<=>B signifie que la liaison existe dans les deux sens.
A=>B=>C equivaut a A=>B ; B=>C
A=>(B;C) equivaut a A=>B ; A=>C
A<=>(B;C) cree des liaisons bidirectionnelles entre A et B, puis A et C

C'est pourquoi les composants sont de la forme :
(Composants)=>Objet_cible
ou, dans cp{}, les composants sont automatiquement relies vers l'objet parent.

Synthax si plusieurs tables :
Si plusieurs tables, il faut preciser le nom de la table avec :
t.nom_table{tab{}}
c.nom_table{col{}}
o.nom_table{obj{}}

ou plus simplement :

nom_table{
col{}
obj{}
}

Tests puissants a faire

  • Une chaine logistique : usine, port, entrepot, client.
  • Une simulation de hausse des prix : production, demande, stock, inflation.
  • Une simulation environnementale : foret, pluie, incendie, secours.
  • Une simulation de discussion : calme, tension, apaisement.
  • Une simulation montrant l'impact économique de la fabriquation et de l'achat d'un objet sur l'environnement
  • Une simulation montrant l'évolution des espèces

Conseils de construction

  • Commencer avec peu d'objets.
  • Verifier d'abord les colonnes et les liaisons.
  • Ajouter ensuite les evenements puis les paternes.
  • Utiliser la page Code seulement quand la structure de base est claire.
